# Glossary — Design System Skill

> **TL;DR:** Terminology reference ensuring consistent language across design system content. Contains shared terms covering spelling, capitalization, punctuation, and preferred/disallowed usage for in-product copy and conversational AI.

## Purpose

Terminology definitions for consistent language across components, themes, copy guidelines, and product UIs.

## Rules

- No invented terms or definitions
- Use consistently across all files
- Synonyms must be explicitly allowed; otherwise disallowed
- Undefined terms → mark as EMPTY, do not define speculatively

---

## AI and Conversational Interface Usage

| Term | Guidance |
|------|----------|
| agent | Can refer to human support agent, AI assistant, or AI agent. Be explicit on first use ("human support agent", "AI agent"). |
| AI | Stands for "artificial intelligence". Do not spell out after first mention. |
| AI-generated | Hyphenate as adjective. Two words as verb. Do not use "gen AI". |
| algorithm / model | Algorithms create models. Do not use interchangeably. |
| answer / response | AI provides a "response" (preferred) or "answer". Use "response" for AI replies. |
| ask / request | A user "asks" a question or makes a "request". "Request" is preferred. |
| assistant | Use for AI services providing assistance through a chat interface. On first mention, use "generative AI assistant" or "AI assistant". |
| bot | Do not use when describing chat-based AI. Use "assistant", "model", or "AI". |
| generative AI | The approved term for referring to the AI experience. Do not abbreviate as "gen AI". |
| generative AI assistant | The approved term for introducing and referring to the AI on first mention. |
| model | A trained AI system. Do not use interchangeably with "algorithm". |
| submit | Use as the label or reference term when a user makes a choice, query, or request. |

## General Writing Usage

| Term | Guidance |
|------|----------|
| abort | Do not use in reference to human beings. Use "stop", "end", or "cancel". |
| above / below | Use only for physical space. For document orientation, use "previous"/"preceding" (not "above") and "following"/"later" (not "below"). |
| activate / deactivate | Use for setting up or disassociating account details. For features, use "enable"/"disable". For settings, use "turn on"/"turn off". |
| ad hoc | Avoid. Use "one-time" instead. |
| affect / effect | "Affect" (verb) means to influence. "Effect" (noun) means result. |
| allow / deny | OK for permissions. Avoid in direct reference to the customer. |
| allowlist / denylist | One word each. Do not use "whitelist" or "blacklist". |
| and/or | Avoid. |
| app / application | Use "app" for mobile software, "application" for all other uses. |
| appear / display / open | Messages "appear". Windows/pages/applications "open". "Display" requires a definite object. |
| as | Do not use "as" to mean "because" — it is frequently ambiguous. |
| backend / frontend | One word each (adjective and noun). |
| be able to | Avoid. Use "can" instead. |
| best | Do not use superlatives. Exception: "best practices" is OK. |
| Boolean | Capitalize unless referring to a specific code element that uses lowercase. |
| choose / select | Preferred over "click" (device-independent language). |

## Punctuation and Formatting Usage

| Term | Guidance |
|------|----------|
| ampersand (&) | Do not use in running text. |
| ellipsis (...) | Do not use in UI copy or instructional text. |
| exclamation point | Do not use. |
| e.g. / i.e. / etc. | Do not use. Write out "for example", "that is", or rephrase. |
| bold | Use only as adjective ("characters in bold type"). Do not use "boldface". |
